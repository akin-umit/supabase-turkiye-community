# Security Policy / Guvenlik Politikasi

Do not open a public issue containing credentials, tokens, private domains, database dumps, or exploit details. Use GitHub private vulnerability reporting when enabled, or contact the repository owner privately.

- Never commit `.env`, dumps, TLS keys, JWT keys, dashboard passwords, or service-role keys.
- Public examples use `example.com` and placeholders.
- Rotate credentials pasted into chat, logs, screenshots, issues, or pull requests.
- Only Kong receives a public route by default.

Security support applies to the latest tagged release. Upstream image vulnerabilities are handled by updating pinned versions after compatibility tests.
